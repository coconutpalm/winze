### Indexing
- A **filesystem watcher** automatically re-indexes files as they are
  created, modified, moved, or deleted. Manual `index_plans` calls are
  only needed for INDEX.md/STATUS.md regeneration.
- **NEVER call `index_plans` with `reset: true` automatically.** A reset

8. **Customization** ‚Äî naming conventions (metadata inference), chunk size
9. **Troubleshooting** ‚Äî new table:

| Problem | Solution |
|---------|----------|
| MCP server shows "failed" after VSCode restart | Check `~/.local/share/winze/plan-server.log` for errors. Kill zombie JVMs: `pkill -f winze-server` then restart |
| Search returns no results | Run `/register-plans` to register the project, then `/index-plans` to build the index |
| Stale results | The filesystem watcher should handle updates automatically. If not, run `/index-plans` for manual reconciliation |
| Multiple JVM processes | `pkill -f winze-server.jar` to kill all, then restart VSCode ‚Äî the proxy auto-starts one clean instance |
| Store corruption (Datalevin errors in log) | `rm -rf ~/.local/share/winze/.datalevin` then restart ‚Äî the index rebuilds from disk files |
| `bb` not found | Install Babashka: `brew install borkdude/brew/babashka` |

## Step 5: Rewrite `setup-prompt.md`

New first-time setup prompt:

```
I need you to set up the clj-llm-memory MCP server for this project.

1. Check prerequisites:
   - JDK 21+ (`java -version`)
   - Babashka (`bb --version`)
   If either is missing, stop and tell me.

2. Build and install:
   cd mcp/winze-server && make install-mcp

3. Restart VS Code (required for MCP server registration).

4. After restart, register this project's Plans directory:
   /register-plans Plans

5. Verify:
   /list-plan-roots    ‚Äî should show this project registered
   /search-plans test  ‚Äî should return results from Plans/
```
